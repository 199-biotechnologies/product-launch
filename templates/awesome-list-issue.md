## Before submitting

1. **Read the target list's `CONTRIBUTING.md` first.** Every awesome list has strict formatting requirements.
2. **Check CoC.** Some lists (awesome-claude-code, some others) **explicitly ban** `gh` CLI / programmatic submissions. Submitting via CLI → automatic close + potential account flag.
3. **Repo must be public, documented, and typically 1+ week old.**
4. **Alphabetical order matters** in most lists. Find your insertion point in the README.

## For lists requiring a PR (no issue template)

1. Fork the repo
2. Find the alphabetical insertion point in the README
3. Add your line in the list's exact format. Common shapes:
   - `- [{{name}}]({{url}}) - {{tagline}}`
   - `- [{{name}}]({{url}}) — {{tagline}}` (em-dash variant)
4. Commit: `Add: {{name}}`
5. PR title: `Add: {{name}}`
6. PR description: 3–5 sentences on what it does + why it belongs + confirmation of CONTRIBUTING.md read

## Lists to consider (by product type)

**oss / cli:**
- `awesome-selfhosted` — strict quality
- `awesome-cli-apps`
- Language-specific: `awesome-python`, `awesome-rust`, `awesome-go`, `awesome-nodejs`

**claude-plugin:**
- `awesome-claude-code` (hesreallyhim) — issue template, CoC bans CLI submission
- `awesome-mcp-servers` if applicable

**ml:**
- `awesome-LLM` / `awesome-generative-ai`
- `awesome-mlops` (if tooling)

**saas / dev tools:**
- `awesome-indie` — maker tools
- `awesome-dev-tools`

## Pace yourself

Submit to 2–3 lists per week, not 10 in one day. Looks organic. Reduces risk of cross-list moderator flagging.
